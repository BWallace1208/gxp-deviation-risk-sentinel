## DOC 03 \- Severity & Escalation Model

## **Severity Definitions**

| *Severity*  | *Description*  |  |
| :---: | :---: | ----- |
| **Low**  | Informational risk condition intended for trend monitoring. |  |
| **Medium**  | Risk condition requiring review but not immediate action. |  |
| **High**  | Significant deviation risk requiring Quality awareness. |  |
| **Critical**  | Immediate deviation risk requiring prompt Quality attention |  |

**Advisory Response Expectations**

| *Severity*  | *Recommended Awareness* |  |
| :---: | :---: | ----- |
| **Low**  | Digital Operations Center / Trend Review |  |
| **Medium**  | Area Leadership  |  |
| **High**  | Quality Assurance & Area Leadership |  |
| **Critical**  | Quality Assurance with escalation |  |

These response expectations are advisory and do not replace established site procedures.

**Severity Override Logic**

Severity may be elevated or reduced by rule logic based on contextual factors such as step criticality, recurrence frequency, or compounding risk signals. Any severity override is recorded as part of the alert metadata.
